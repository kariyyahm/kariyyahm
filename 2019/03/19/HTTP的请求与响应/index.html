HTTP 的请求与响应
2019-03-19

HTTP 请求

请求格式

1 动词 路径 协议/版本
2 Key1: value1
2 Key2: value2
2 Key3: value3
2 Content-Type: application/x-www-form-urlencoded
2 Host: www.baidu.com
2 User-Agent: curl/7.54.0
3
4 要上传的数据

需要注意的是：

  • 请求最多包含四部分，最少包含三部分。（也就是说第四部分可以为空）
  • 第三部分永远都是一个回车（\n）
  • 动词有 GET POST PUT PATCH DELETE 等
  • 这里的路径包括「查询参数」，但不包括「锚点」
  • 如果你没有写路径，那么路径默认为 /
  • 第 2 部分中的 Content-Type 标注了第 4 部分的格式

如何使用开发者工具查看请求

  1. 打开浏览器并按 F12
  2. 地址栏输入网址
  3. 在 Network 点击，查看 Request Headers
  4. 点击「view source」
  5. 如果有请求的第四部分，那么在 FormData 或 Payload 里面可以看到

HTTP 响应

响应格式

1 协议/版本号 状态码 状态解释
2 Key1: value1
2 Key2: value2
2 Content-Length: 17931
2 Content-Type: text/html
3
4 要下载的内容

需要注意的是：

  • 状态码要背，是服务器对浏览器说的话
    • 1xx 表示消息
    • 2xx 表示成功
    • 3xx 表示重定向
    • 4xx 表示客户端错误
    • 5xx 表示服务器错误
  • 第2部分中的 Content-Type 标注了第 4 部分的格式
  • 第2部分中的 Content-Type 遵循 MIME 规范

如何使用开发者工具查看相应

  1. 打开浏览器并按 F12
  2. 地址栏输入网址
  3. 在 Network 点击，查看 Response Headers
  4. 点击「view source」
  5. 会看到响应的前两部分
  6. 查看 Response 或者 Preview，会看到响应的第 4 部分
